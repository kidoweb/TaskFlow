# Инструкция по деплою на GitHub Pages

## Автоматический деплой через GitHub Actions (Рекомендуется)

1. **Настройте репозиторий:**
   - Убедитесь, что ваш код находится в ветке `main` или `master`
   - Перейдите в Settings → Pages вашего репозитория
   - В разделе "Source" выберите "GitHub Actions"

2. **Base path настроен автоматически:**
   - Для репозитория `kidoweb/TaskFlow` base path установлен как `/TaskFlow/`
   - Это настроено в `vite.config.ts` и `.github/workflows/deploy.yml`
   - Если нужно изменить, отредактируйте эти файлы

3. **Настройте Firebase:**
   - Убедитесь, что в `firebase.ts` указаны правильные credentials
   - Настройте Firestore Security Rules для публичного доступа (если нужно)

4. **Запустите деплой:**
   - Просто запушьте изменения в ветку `main`:
   ```bash
   git add .
   git commit -m "Prepare for deployment"
   git push origin main
   ```
   - GitHub Actions автоматически соберет и задеплоит проект

5. **Проверьте результат:**
   - После завершения workflow ваш сайт будет доступен по адресу:
   - `https://kidoweb.github.io/TaskFlow/`

## Ручной деплой через gh-pages

1. **Установите зависимости:**
   ```bash
   npm install
   ```

2. **Base path уже настроен:**
   - Для репозитория `TaskFlow` base path установлен как `/TaskFlow/`
   - Если нужно изменить, отредактируйте `vite.config.ts`

3. **Выполните деплой:**
   ```bash
   npm run deploy
   ```

4. **Настройте GitHub Pages:**
   - Перейдите в Settings → Pages вашего репозитория
   - В разделе "Source" выберите ветку `gh-pages` и папку `/ (root)`
   - Сохраните изменения

## Важные замечания

- **Base Path:** Если ваш репозиторий не называется `username.github.io`, обязательно укажите правильный base path в `vite.config.ts`
- **Firebase:** Убедитесь, что ваши Firebase credentials настроены правильно и доступны публично
- **Environment Variables:** Если используете переменные окружения, они должны быть доступны во время сборки
- **Router:** React Router настроен на использование `HashRouter`, что идеально работает с GitHub Pages без необходимости настройки сервера

## Проверка после деплоя

1. Откройте ваш сайт на GitHub Pages
2. Проверьте, что все маршруты работают корректно
3. Убедитесь, что Firebase подключение работает
4. Проверьте, что все ресурсы (изображения, стили) загружаются правильно
